````markdown
# Last Unique Touch — Design Rules (Do Not Violate)

> هذه القواعد مستخلصة من تقرير Tri-AI v3.0. أي مخالفة لها = bug مؤكد.

## 1. i18n — لا نصوص hardcoded
- ❌ لا تستخدم `'KWD/day'`, `'KWD'`, `'Subject'`, أو أي نص آخر hardcoded في JSX
- ✅ كل النصوص تمر عبر `t('key')` من `next-intl`
- ✅ كل مفتاح في `ar.json` يجب أن يكون في `en.json` أيضاً
- ✅ Admin panel也不例外 — كل النصوص مترجمة

## 2. الألوان — مصدر واحد للحقيقة
- ❌ لا تخترع hex codes جديدة في JSX
- ✅ استخدم CSS variables من `globals.css` (`--c-ink`, `--c-paper`, `--c-brand`, إلخ)
- ❌ لا تكرر نفس اللون في 3 أنظمة متغيرات (`--c-gold` + `--primary` + `--color-gold`)
- ✅ استخدم متغيراً مصدراً واحداً (`--c-brand`) واجعل البقية يشيرون إليه

## 3. لغة التصميم — توحيد صارم
- ❌ لا تستخدم `rounded-xl` في صفحة و `borderRadius: '2px'` في صفحة أخرى لنفس العنصر
- ✅ اختر لغة واحدة لكل البطاقات/الكروت وطبّقها في كل مكان
- ❌ لا تستخدم `font-display` لعنوان في صفحة و `font-semibold` لنفس العنصر في صفحة أخرى
- ✅ عناوين المنتجات موحّدة عبر كل الصفحات

## 4. RTL — احترام الاتجاه
- ❌ لا تستخدم `left`/`right` في CSS أو `x: '100%'` ثابت في animations
- ✅ استخدم logical properties: `ms-`, `me-`, `ps-`, `pe-`, `start-`, `end-`
- ✅ animations تعتمد على `locale`: `const drawerX = locale === 'ar' ? '-100%' : '100%'`
- ✅ استخدم `ArrowLeft`/`ArrowRight` من lucide-react مع تبديل حسب الـ locale

## 5. التصفّح (Navigation) — استخدم Next.js Link
- ❌ لا تستخدم `<a href>` للروابط الداخلية
- ✅ استخدم `Link` من `@/i18n/routing` (يحافظ على locale + client-side navigation)
- ✅ MagneticButton وكل wrapper components يجب أن تستخدم `Link` داخلياً

## 6. الصور — استخدم next/image دائماً
- ❌ لا تستخدم `<img>` أبداً (لأسباب أداء + SEO)
- ✅ استخدم `next/image` مع `width`, `height`, `alt`, و `sizes`
- ✅ alt text وصفي للصور المعلوماتية، `alt=""` للزخرفية
- ❌ لا تمرر `undefined` لـ `src` — استخدم conditional rendering

## 7. الأزرار التفاعلية — cursor صحيح
- ❌ لا تستخدم `cursor-pointer` على عناصر غير قابلة للنقر
- ✅ `cursor-pointer` فقط للعناصر القابلة للنقر فعلاً
- ✅ `cursor-default` أو `cursor-not-allowed` للعناصر المعطّلة

## 8. A11y — قواعد إلزامية
- ✅ كل `<button>` بأيقونة فقط يجب أن يكون له `aria-label`
- ✅ كل `<input>` يجب أن يكون له `<label>` مرتبط (via `htmlFor` و `id`)
- ✅ Toast container: `role="region"` + `aria-label="Notifications"`
- ✅ Toast فردي: `role="status"` (نجاح) أو `role="alert"` (خطأ) + `aria-live`
- ✅ Touch targets: `min-h-[44px] min-w-[44px]` (WCAG 2.5.8)
- ✅ Form errors: `role="alert"` + icon + رسالة محددة (ليس اسم الحقل مكرراً)
- ✅ Pagination: `aria-label={t('products.pagination')}` + `aria-hidden` للـ ellipsis
- ✅ Form fields: `autoComplete` صحيح (`cc-number`, `cc-exp`, `cc-csc`, `email`, `tel`)
- ✅ H1 واحد لكل صفحة
- ✅ Skip to content link في كل layout

## 9. React Hook Form + Radix — استخدم Controller
- ❌ لا تستخدم `{...register('field')}` على Radix components (Checkbox, Switch, Select)
- ✅ استخدم `<Controller control={control} name="field" render={...} />`
- ✅ Radix يستخدم `checked` + `onCheckedChange`، وليس `value` + `onChange`

## 10. Skeletons + Empty States — توحيد
- ❌ لا تستخدم `<div className="animate-pulse">...</div>` كـ loading state
- ✅ استخدم skeleton components (`CartSkeleton`, `CheckoutSkeleton`, `ProductsGridSkeleton`)
- ✅ استخدم `<EmptyState>` الموحّد لكل empty states (icon + title + description + action)

## 11. الكود الميت — احذفه فوراً
- ❌ لا تترك components غير مستوردة (مثل `brand-selector.tsx` سابقاً)
- ✅ ابحث بـ `grep -r "ComponentName" src/` قبل الحذف للتأكد
- ✅ احذف الـ CSS الميت (كتل `dark {...}` بدون تفعيل، tokens مكررة، إلخ)

## 12. Typography — أوزان محدودة
- ❌ لا تحمّل 5+ أوزان لكل خط (يبطّئ التحميل)
- ✅ Tajawal: `['400', '700']` فقط
- ✅ Cormorant: `['400', '500']` فقط
- ✅ Inter: `['400', '500', '700']` فقط

## 13. Three.js — أداء
- ✅ استخدم `frameloop="demand"` لإيقاف rAF عند عدم الحاجة
- ✅ استخدم `dynamic` مع `ssr: false` للـ lazy load
- ✅ وفّر `fallback` (مثل `StaticGradientBg`) عند فشل WebGL
- ✅ استخدم `IntersectionObserver` لإيقاف الـ canvas offscreen

## 14. Dead Code — لا ternaries بنفس القيمة
- ❌ لا تكتب `cond ? 'a' : 'a'` (نفس القيمة في الفرعين)
- ✅ راجع كل ternary — إن كانت القيمتان متطابقتين، احذف الـ ternary

## 15. الإصدار — اختبار قبل الإصدار
- ✅ `bun run lint` — 0 errors
- ✅ `bun run typecheck` — 0 errors
- ✅ `bun run test` — كل الاختبارات تنجح
- ✅ `bun run build` — ينجح بدون MISSING_MESSAGE
- ✅ اختبر بصرياً على mobile (375px) + tablet (768px) + desktop (1280px+)
- ✅ اختبر RTL + LTR
```

#### 6E. إنشاء Script للتحقق التلقائي

```typescript
#!/usr/bin/env bun
/**
 * Design Rules Checker — يتحقق من القواعد في DESIGN_RULES.md
 * شغّل: bun scripts/check-design-rules.ts
 */

import { glob } from 'glob';
import * as fs from 'fs';

const errors: string[] = [];

// 1. لا <img> — استخدم next/image
const files = await glob('src/**/*.{ts,tsx}');
for (const file of files) {
  const content = fs.readFileSync(file, 'utf-8');
  
  // Check for <img>
  if (/<img\s/.test(content)) {
    errors.push(`❌ ${file}: استخدم next/image بدلاً من <img>`);
  }
  
  // Check for hardcoded currency
  if (/['"]KWD['"]/.test(content) || /['"]د\.ك['"]/.test(content)) {
    errors.push(`❌ ${file}: استخدم t("common.currency") بدلاً من hardcoded currency`);
  }
  
  // Check for <a href> (internal links)
  if (/<a\s+href=['"]\/(?!tel:|mailto:|https?:)/.test(content)) {
    errors.push(`❌ ${file}: استخدم Link من @/i18n/routing بدلاً من <a href> للروابط الداخلية`);
  }
  
  // Check for hardcoded Arabic in JSX (basic check)
  const arabicInJsx = content.match(/>\s*[\u0600-\u06FF][^<]*</g);
  if (arabicInJsx && !file.includes('test') && !file.includes('locale') && !file.includes('messages')) {
    errors.push(`⚠️ ${file}: قد يحتوي على نصوص عربية hardcoded — تحقق من i18n`);
  }
  
  // Check for dead ternaries (same value in both branches)
  const deadTernaries = content.match(/\?\s*['"]([^'"]+)['"]\s*:\s*['"]\1['"]/g);
  if (deadTernaries) {
    errors.push(`❌ ${file}: dead ternary detected — نفس القيمة في الفرعين: ${deadTernaries.join(', ')}`);
  }
  
  // Check for left/right instead of logical properties
  if (/\bml-|\bmr-|\bpl-|\bpr-|\bleft-|\bright-/.test(content)) {
    errors.push(`⚠️ ${file}: استخدم logical properties (ms-, me-, ps-, pe-, start-, end-) بدلاً من left/right`);
  }
}

if (errors.length > 0) {
  console.error('\n🚨 Design Rules Violations:\n');
  errors.forEach(e => console.error(e));
  console.error(`\n❌ ${errors.length} violation(s) found. اقرأ DESIGN_RULES.md.`);
  process.exit(1);
} else {
  console.log('✅ All design rules passed!');
}
```
````
