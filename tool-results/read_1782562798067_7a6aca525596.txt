    63→## 2. منهجية Vibe Coding 7.0
    64→
    65→### 2.1 ما هي منهجية Vibe Coding 7.0؟
    66→
    67→هي منهجية تطوير برمجي تعتمد على ثلاث ركائز أساسية تضمن جودة الكود وأمانه وقابلية صيانته عند بناء المشاريع باستخدام وكلاء الذكاء الاصطناعي. هذه المنهجية طُوّرت عبر 13 سبنت (Sprint) من التطوير المتواصل، وقد أثبتت فعاليتها في تقليل الأخطاء الحرجة بنسبة 60% بعد تطبيقها بشكل صحيح.
    68→
    69→### 2.2 الركيزة الأولى: مبدأ المصدر الواحد (Single Source Principle)
    70→
    71→**المفهوم:** يجب أن يكون هناك وكيل ذكاء اصطناعي واحد فقط هو "المهندس المعماري" للمشروع، يجمع كل القرارات المعمارية والقواعد في ملف واحد قابل للوصول من جميع الوكلاء.
    72→
    73→**التطبيق العملي:**
    74→
    75→- أنشئ ملف `ARCHITECTURE.md` في جذر المشروع يحتوي على كل القواعد.
    76→- كل وكيل فرعي (subagent) يجب أن يقرأ هذا الملف قبل البدء بأي مهمة.
    77→- لا تسمح لوكلاء متعددين باتخاذ قرارات معمارية متضاربة.
    78→- أي قرار جديد يضاف لملف `ARCHITECTURE.md` ويُوثّق بتاريخه وسببه.
    79→
    80→**قالب `ARCHITECTURE.md`:**
    81→
    82→```markdown
    83→# ARCHITECTURE.md — قرارات معمارية للمشروع
    84→
    85→## Tech Stack
    86→- Next.js 16 (App Router)
    87→- TypeScript (strict mode)
    88→- Clerk for Authentication
    89→- Prisma + PostgreSQL
    90→- Tailwind CSS v4
    91→- shadcn/ui components
    92→
    93→## Rules
    94→- Rule 1: Prisma singleton pattern (lib/prisma.ts)
    95→- Rule 2: Clerk auth pattern (auth() + clerkMiddleware)
    96→- Rule 3: API response format (successResponse/errorResponse)
    97→- Rule 4: Decimal → .toString() before JSON
    98→- Rule 5: kebab-case file names
    99→- Rule 6: CSS Logical Properties (RTL-first)
   100→- Rule 7: shadcn/ui only (no Material UI/Ant Design)
   101→- Rule 8: useTranslations() for all UI text
   102→- Rule 9: Zod v4 with `message` for every field
   103→- Rule 10: catch blocks use `unknown` (not `any`)
   104→- Rule 11: Role protection on server side
   105→- Rule 12: No `any` type (use `unknown` or explicit interfaces)
   106→- Rule 13: No Server Actions — use API Routes
   107→- Rule 14: Link from `@/i18n/routing`
   108→- Rule 15: `@/` import alias only
   109→
   110→## Decisions Log
   111→- 2026-06-15: اخترنا Clerk بدلاً من NextAuth لتوفير أسابيع من العمل
   112→- 2026-06-16: استخدمنا Decimal(10, 3) للعملة الكويتية (KWD = 1000 فلس)
   113→- 2026-06-17: اعتمدنا SQLite للتطوير المحلي و PostgreSQL للإنتاج
   114→```
   115→
   116→### 2.3 الركيزة الثانية: حلقة Micro-Sprint (Micro-Sprint Loop)
   117→
   118→**المفهوم:** بدلاً من العمل بسبنتات طويلة (أسبوعين أو أكثر)، قسّم العمل إلى سبنتات صغيرة جداً (30 دقيقة إلى 3 ساعات لكل سبنت) مع دورة تطوير سريعة.
   119→
   120→**هيكل السبنت الواحد:**
   121→
   122→```
   123→1. تحديد الهدف الواحد للسبنت (Goal)
   124→2. كتابة الكود (Code)
   125→3. الاختبار الفوري (Test)
   126→4. المراجعة الذاتية (Self-Review)
   127→5. التحقق من القواعد (Rules Check)
   128→6. الدمج (Merge)
   129→```
   130→
   131→**الفوائد المثبتة:**
   132→
   133→- اكتشاف الأخطاء في وقت مبكر جداً.
   134→- تقليل التراجع عن القرارات (rework) بنسبة 70%.
   135→- الحفاظ على تركيز الوكيل على مهمة واحدة.
   136→- إمكانية العودة لنسخة سابقة بسهولة عند الفشل.
   137→
   138→### 2.4 الركيزة الثالثة: Stop Gate QA (بوابة التوقف لضمان الجودة)
   139→
   140→**المفهوم:** في نهاية كل سبنت، يجب التوقف عند "بوابة جودة" لا يمكن تجاوزها إلا بعد التحقق من معايير محددة (راجع قسم 12 في هذا الملف).
   141→
   142→> ⚠️ **تحذير حرج:** لا تتجاوز بوابة الجودة أبداً. إذا فشل أي بند، أصلحه قبل المضي قدماً. تجاوز البوابة هو السبب الأول لتراكم الديون التقنية.
   143→
   144→### 2.5 دورة حياة المشروع الكاملة
   145→
   146→```
   147→[التخطيط] → [الإعداد] → [السبنت 1] → [Stop Gate] → [السبنت 2] → [Stop Gate] →
   148→   ... → [التدقيق النهائي] → [الإطلاق] → [المراقبة]
   149→```
   150→
   151→كل مرحلة لها مدخلات ومخرجات محددة، ولا يمكن الانتقال للمرحلة التالية إلا بإكمال مخرجات المرحلة السابقة.

   155→## 3. المكدس التقني القياسي (Tech Stack)

   158→
   159→| الطبقة | التقنية | السبب |
   160→|--------|---------|-------|
   161→| **Framework** | Next.js 16 (App Router) | أحدث إصدار، SSR/SSG، RSC، API Routes |

   167→| **Database (Dev)** | SQLite | تطوير سريع، لا حاجة لخادم |
   168→| **Database (Prod)** | PostgreSQL (Supabase) | موثوقية، JSONB، Enums، pooling |

   171→| **State (Client)** | Zustand | بساطة وأداء |
   172→| **State (Server)** | TanStack Query | caching، optimistic updates |
   173→| **Validation** | Zod v4 | تحقق من الأنواع في الـ runtime |
   174→| **Animation** | Framer Motion | حركات ناعمة |
   175→| **3D (اختياري)** | Three.js + React Three Fiber | تأثيرات بصرية متقدمة |
   176→| **Package Manager** | Bun | سرعة فائقة + دعم TypeScript أصلي |
   177→| **Runtime** | Bun | أداء أفضل من Node.js |

   180→| **Hosting** | Railway | نشر سهل + healthcheck + variables |
   181→| **Database Hosting** | Supabase | طبقة مجانية + PgBouncer pooling |
   182→
   183→### 3.2 البنية التحتية
   184→
   185→```
   186→Next.js App (port 3000)
   187→    │
   188→    ├── Caddy Gateway (port 81 → exposed externally)
   189→    │       └── XTransformPort query → mini-services
   190→    │
   191→    ├── Mini-Services (WebSocket, cron, etc.)
   192→    │       └── port 3003+ (each independent Bun project)
   193→    │
   194→    └── Database (SQLite dev / PostgreSQL prod)
   195→```
   196→
   197→### 3.3 لماذا هذه الاختيارات؟
   198→
   199→1. **SQLite للتطوير**: لا حاجة لخادم قاعدة بيانات، سريع، كافٍ لمعظم المشاريع. انتقل إلى PostgreSQL للإنتاج.
   200→2. **Clerk بدلاً من بناء auth من الصفر**: يوفّر أسابيع من العمل، يدعم متعدد المزودين، أمان جاهز.
   201→3. **shadcn/ui بدلاً من مكتبات UI الجاهزة**: الكود يصبح ملكك، قابل للتخصيص الكامل، لا قفل بائع.
   202→4. **Zustand بدلاً من Redux**: أبسط، أسرع، كافٍ لمعظم الحالات.
   203→5. **next-intl بدلاً من next-i18next**: مدمج أفضل مع App Router.
   204→6. **Bun بدلاً من npm/pnpm**: أسرع بـ 10-30 مرة في تثبيت الحزم، يدعم TypeScript أصلاً بدون `tsx`.

   208→## 4. القواعد الصارمة الـ15 (غير قابلة للتفاوض)
   209→
   210→> ⚠️ تطبيق هذه القواعد من البداية يمنع 90% من الأخطاء. تجاهل أي منها سيؤدي إلى أخطاء صعبة الإصلاح لاحقاً.
   211→
   212→### القاعدة 1: Prisma Singleton Pattern
   213→
   214→**المشكلة:** في التطوير، يحفظ Next.js الـ module، مما يُنشئ عدة اتصالات ويسبب `PrismaClientInitializationError` واستنفاد الاتصالات.
   215→
   216→```typescript
   217→// src/lib/db.ts
   218→import { PrismaClient } from '@prisma/client'
   219→
   220→const globalForPrisma = globalThis as unknown as {
   221→  prisma: PrismaClient | undefined
   222→}
   223→
   224→export const db =
   225→  globalForPrisma.prisma ??
   226→  new PrismaClient({
   227→    log: process.env.NODE_ENV === 'development' ? ['warn', 'error'] : ['error'],
   228→  })
   229→
   230→if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = db
   231→```
   232→
   233→**الاستخدام:** `import { db } from '@/lib/db'` — دائماً، في كل مكان.
   234→
   235→### القاعدة 2: مصادقة آمنة — `useSafeAuth()` و `useSafeUser()`
   236→
   237→**المشكلة:** `useUser()` من Clerk يُرجع `undefined` أثناء التحميل، مما يسبب crashes في الواجهة.
   238→
   239→```typescript
   240→// src/hooks/use-safe-auth.ts
   241→import { useAuth, useUser } from '@clerk/nextjs'
   242→
   243→export function useSafeAuth() {
   244→  const { isLoaded, isSignedIn, userId } = useAuth()
   245→  return {
   246→    isLoaded,
   247→    isSignedIn: isSignedIn ?? false,
   248→    userId: userId ?? null,
   249→  }
   250→}
   251→
   252→export function useSafeUser() {
   253→  const { user } = useUser()
   254→  return {
   255→    name: user?.firstName ?? '',
   256→    email: user?.emailAddresses?.[0]?.emailAddress ?? '',
   257→    imageUrl: user?.imageUrl ?? '',
   258→  }
   259→}
   260→```
   261→
   262→**في Server Components:**
   263→
   264→```typescript
   265→import { auth, currentUser } from '@clerk/nextjs/server'
   266→
   267→export async function getCurrentUser() {
   268→  const { userId } = await auth()
   269→  if (!userId) return null
   270→
   271→  const clerkUser = await currentUser()
   272→  if (!clerkUser) return null
   273→
   274→  // مزامنة مع قاعدة البيانات
   275→  return await db.user.findUnique({ where: { clerkId: userId } })
   276→}
   277→```

   279→### القاعدة 3: استجابات API موحّدة
   280→
   281→```typescript
   282→// src/lib/api-response.ts
   283→export interface PaginationMeta {
   284→  page: number
   285→  limit: number
   286→  total: number
   287→  totalPages: number
   288→}
   289→
   290→export function successResponse<T>(
   291→  data: T,
   292→  message: string,
   293→  meta?: PaginationMeta,
   294→  status = 200
   295→) {
   296→  return Response.json(
   297→    { success: true, data, message, ...(meta && { meta }) },
   298→    { status }
   299→  )
   300→}
   301→
   302→export function errorResponse(
   303→  code: string,
   304→  message: string,
   305→  field?: string,
   306→  status = 400
   307→) {
   308→  return Response.json(
   309→    { success: false, error: { code, message, field: field ?? null } },
   310→    { status }
   311→  )
   312→}
   313→```
   314→
   315→**التنسيق الموحّد:**
   316→
   317→```json
   318→// نجاح
   319→{ "success": true, "data": {...}, "message": "...", "meta": {...} }
   320→
   321→// خطأ
   322→{ "success": false, "error": { "code": "...", "message": "...", "field": null } }
   323→```
   324→
   325→### القاعدة 4: Decimal → `.toString()` دائماً
   326→
   327→عند إرجاع قيم Decimal من Prisma في JSON، يجب تحويلها إلى string دائماً لتجنب فقدان الدقة وأخطاء التسلسل.
   328→
   329→```typescript
   330→// ❌ خطأ: قد يفقد الدقة + يفشل JSON.stringify
   331→return successResponse({ price: event.price }, '...')
   332→
   333→// ✅ صحيح
   334→return successResponse({ price: event.price.toString() }, '...')
   335→```
   336→
   337→**لـ SQLite (الذي لا يدعم Decimal):** خزّنها كـ String بـ 3 منازل عشرية:
   338→
   339→```prisma
   340→price String  // "15.500" بدلاً من Decimal
   341→```
   342→
   343→**لـ PostgreSQL:** استخدم `Decimal(10, 3)` للعملة الكويتية (الدينار = 1000 فلس):
   344→
   345→```prisma
   346→price Decimal @db.Decimal(10, 3)
   347→```

   366→### القاعدة 6: CSS Logical Properties (RTL-first)
   367→
   368→صمم وأنت تفكر في RTL أولاً، لأن العربية هي اللغة الأساسية. اختبر RTL قبل LTR.
   369→
   370→```css
   371→/* ❌ خطأ: لا يدعم RTL */
   372→margin-left: 16px;
   373→padding-right: 8px;
   374→text-align: left;
   375→.sidebar { left: 0; }
   376→.icon { margin-right: 8px; }
   377→
   378→/* ✅ صحيح: يدعم RTL تلقائياً */
   379→margin-inline-start: 16px;
   380→padding-inline-end: 8px;

   437→### القاعدة 10: catch blocks تستخدم `unknown` وليس `any`
   438→
   439→```typescript
   440→// ❌ خطأ
   441→try {
   442→  // ...
   443→} catch (error: any) {
   444→  console.log(error.message)
   445→  return errorResponse('ERR', error.message)
   446→}
   447→
   448→// ✅ صحيح
   449→try {
   450→  // ...
   451→} catch (error: unknown) {
   452→  const message = error instanceof Error ? error.message : 'حدث خطأ غير متوقع'
   453→  console.error('API error:', message, error)
   454→  return errorResponse('INTERNAL_ERROR', message, undefined, 500)
   455→}

   458→### القاعدة 11: حماية الأدوار على مستوى الخادم
   459→
   460→تحقق من دور المستخدم في كل صفحة محمية وAPI endpoint، وليس فقط من تسجيل الدخول.
   461→
   462→```typescript
   463→// src/app/api/v1/admin/users/route.ts
   464→import { getCurrentUser } from '@/lib/auth-server'
   465→
   466→export async function GET() {
   467→  try {
   468→    const user = await getCurrentUser()
   469→    if (!user) {
   470→      return errorResponse('UNAUTHORIZED', 'يجب تسجيل الدخول', undefined, 401)
   471→    }
   472→    if (user.role !== 'ADMIN') {
   473→      return errorResponse('FORBIDDEN', 'غير مصرح', undefined, 403)
   474→    }
   475→    // ... المنطق
   476→  } catch (error: unknown) {
   477→    // ...
   478→  }
   479→}
   480→```
   481→
   482→**في middleware:**
   483→
   484→```typescript
   485→// src/middleware.ts
   486→import { clerkMiddleware, createRouteMatcher } from '@clerk/nextjs/server'
   487→
   488→const isAdminRoute = createRouteMatcher(['/admin(.*)'])
   489→const isOrganizerRoute = createRouteMatcher(['/organizer(.*)'])
   490→const isProtected = createRouteMatcher(['/dashboard(.*)', '/api/v1/(.*)'])
   491→
   492→export default clerkMiddleware(async (auth, req) => {
   493→  if (isAdminRoute(req)) {
   494→    await auth.protect({ role: 'org:admin' })
   495→  }
   496→  if (isOrganizerRoute(req)) {
   497→    await auth.protect({ role: 'org:member' })
   498→  }
   499→  if (isProtected(req)) {
   500→    await auth.protect()
   501→  }
   502→})
   503→```

   505→### القاعدة 12: لا `any` نهائياً
   506→
   507→**البدائل:**
   508→
   509→- `unknown` عندما لا تعرف النوع.
   510→- `Record<string, unknown>` للكائنات العامة.
   511→- تعريف `type` أو `interface` واضح.
   512→
   513→```typescript
   514→// ❌ خطأ
   515→function processData(data: any) {
   516→  return data.users
   517→}
   518→
   519→// ✅ صحيح
   520→interface UserData {
   521→  users: User[]
   522→}
   523→
   524→function processData(data: unknown): User[] {
   525→  if (typeof data !== 'object' || data === null) {
   526→    throw new Error('Invalid data')
   527→  }
   528→  if (!('users' in data)) {
   529→    throw new Error('Missing users field')
   530→  }
   531→  return (data as UserData).users
   532→}
   533→```

   535→### القاعدة 13: لا Server Actions — استخدم API Routes
   536→
   537→كل منطق الخادم يكون في `src/app/api/v1/.../route.ts`. الواجهة تستدعيها عبر `fetch` أو `api-client.ts`.
   538→
   539→**الأسباب:**
   540→
   541→- API Routes أسهل في الاختبار.
   542→- تجنب مشاكل CSRF الافتراضية في Server Actions.
   543→- فصل أوضح بين client و server.
   544→- توثيق API أوضح.

   694→

   696→
   697→- **الجمهور المستهدف**: من سيستخدم المنصة؟ ما احتياجاته؟

   701→- **القيود التجارية**: ما الميزانية؟ ما الجدول الزمني؟
   702→
   703→**ثم:**
   704→
   705→1. أنشئ repository مع `.gitignore` صحيح.
   706→2. أعد branching strategy (`main`, `develop`, `feature/*`).

  1078→### 8.1 Prisma Schema — نموذج مرجعي
  1079→
  1080→```prisma
  1081→// prisma/schema.prisma
  1082→generator client {
  1083→  provider = "prisma-client-js"
  1084→}
  1085→
  1086→datasource db {
  1087→  provider = "postgresql"  // أو "sqlite" للتطوير المحلي
  1088→  url      = env("DATABASE_URL")
  1089→  directUrl = env("DIRECT_URL")  // مهم للـ migrations مع PostgreSQL
  1090→}
  1091→
  1092→// Enums (PostgreSQL فقط — SQLite لا يدعمها)
  1093→enum EventStatus {
  1094→  DRAFT
  1095→  PUBLISHED
  1096→  CANCELLED
  1097→  COMPLETED
  1098→}
  1099→
  1100→enum UserRole {
  1101→  USER
  1102→  ORGANIZER
  1103→  ADMIN
  1104→}
  1105→
  1106→enum Locale {
  1107→  ar
  1108→  en
  1109→}
  1110→
  1111→// النماذج
  1112→model User {
  1113→  id        String   @id @default(cuid())
  1114→  clerkId   String   @unique
  1115→  email     String   @unique
  1116→  name      String
  1117→  role      UserRole @default(USER)
  1118→  isActive  Boolean  @default(true)
  1119→
  1120→  // Relations
  1121→  events    Event[]
  1122→
  1123→  createdAt DateTime @default(now())
  1124→  updatedAt DateTime @updatedAt
  1125→
  1126→  @@index([email])
  1127→  @@map("users")
  1128→}
  1129→
  1130→model Event {
  1131→  id          String      @id @default(cuid())
  1132→  slug        String      @unique
  1133→  status      EventStatus @default(DRAFT)
  1134→  startDate   DateTime
  1135→  endDate     DateTime
  1136→  organizerId String
  1137→  organizer   User        @relation(fields: [organizerId], references: [id])
  1138→  categoryId  String
  1139→  category    Category    @relation(fields: [categoryId], references: [id])
  1140→
  1141→  // ترجمات JSONB (بديل أفضل من جدول منفصل)
  1142→  title       Json        // { "ar": "...", "en": "..." }
  1143→  description Json
  1144→
  1145→  // الحقول المالية — استخدم Decimal(10, 3) للعملة الكويتية
  1146→  minPrice    Decimal?    @db.Decimal(10, 3)
  1147→
  1148→  ticketTiers TicketTier[]
  1149→  createdAt   DateTime    @default(now())
  1150→  updatedAt   DateTime    @updatedAt
  1151→
  1152→  @@index([status, startDate])
  1153→  @@index([categoryId])
  1154→  @@index([organizerId])
  1155→  @@map("events")
  1156→}
  1157→
  1158→model TicketTier {
  1159→  id        String  @id @default(cuid())
  1160→  eventId   String
  1161→  event     Event   @relation(fields: [eventId], references: [id], onDelete: Cascade)
  1162→  name      Json    // { "ar": "...", "en": "..." }
  1163→  price     Decimal @db.Decimal(10, 3)
  1164→  quantity  Int
  1165→  sold      Int     @default(0)
  1166→
  1167→  @@index([eventId])
  1168→  @@map("ticket_tiers")
  1169→}
  1170→
  1171→model Category {
  1172→  id    String  @id @default(cuid())
  1173→  slug  String  @unique
  1174→  name  Json    // { "ar": "...", "en": "..." }
  1175→  events Event[]
  1176→
  1177→  @@map("categories")
  1178→}
  1179→```
  1180→
  1181→**دروس مهمة من الـ schema:**
  1182→
  1183→- استخدم `Decimal(10, 3)` للعملة الكويتية (الدينار = 1000 فلس)، وليس `Decimal(10, 2)`.
  1184→- استخدم `Json` لحقل الترجمات متعددة اللغات بدلاً من جدول منفصل.
  1185→- أضف `@@index` على الحقول المستخدمة بكثرة في `WHERE` و `ORDER BY`.
  1186→- استخدم `onDelete: Cascade` بحذر — فقط عندما يكون منطقياً (مثل حذف تذاكر الفعالية عند حذف الفعالية).
  1187→- استخدم `@map("snake_case")` لأسماء الأعمدة، `@@map("plural_table")` لأسماء الجداول.
  1188→- استخدم `cuid()` للـ IDs (أفضل من UUID للـ URLs).
  1189→- **SQLite لا يدعم `enum` و `Decimal`** — خزّنها كـ `String`.

  1216→### 8.3 API Route — قالب قياسي
  1217→
  1218→```typescript
  1219→// src/app/api/v1/events/route.ts
  1220→import { NextRequest } from 'next/server'
  1221→import { db } from '@/lib/db'
  1222→import { successResponse, errorResponse } from '@/lib/api-response'
  1223→import { getCurrentUser } from '@/lib/auth-server'
  1224→import { eventCreateSchema } from '@/lib/validators/event-schema'
  1225→import { z } from 'zod'
  1226→
  1227→const querySchema = z.object({
  1228→  page: z.coerce.number().min(1).default(1),
  1229→  limit: z.coerce.number().min(1).max(50).default(10),
  1230→  locale: z.enum(['ar', 'en']).default('ar'),
  1231→  status: z.enum(['DRAFT', 'PUBLISHED', 'CANCELLED', 'COMPLETED']).optional(),
  1232→})
  1233→
  1234→export async function GET(req: NextRequest) {
  1235→  try {
  1236→    // 1. المصادقة
  1237→    const currentUser = await getCurrentUser()
  1238→    if (!currentUser) {
  1239→      return errorResponse('UNAUTHORIZED', 'يجب تسجيل الدخول', undefined, 401)
  1240→    }
  1241→
  1242→    // 2. التحقق من المدخلات
  1243→    const { searchParams } = new URL(req.url)
  1244→    const { page, limit, locale, status } = querySchema.parse(
  1245→      Object.fromEntries(searchParams)
  1246→    )
  1247→
  1248→    // 3. الاستعلام
  1249→    const where = {
  1250→      ...(status && { status }),
  1251→    }
  1252→
  1253→    const [events, total] = await Promise.all([
  1254→      db.event.findMany({
  1255→        where,
  1256→        skip: (page - 1) * limit,
  1257→        take: limit,
  1258→        orderBy: { startDate: 'asc' },
  1259→        include: {
  1260→          category: true,
  1261→          ticketTiers: { select: { id: true, price: true, quantity: true, sold: true } },
  1262→        },
  1263→      }),
  1264→      db.event.count({ where }),
  1265→    ])
  1266→
  1267→    // 4. تحويل Decimal إلى String (القاعدة 4)
  1268→    const safeEvents = events.map((event) => ({
  1269→      ...event,
  1270→      minPrice: event.minPrice?.toString() ?? null,
  1271→      ticketTiers: event.ticketTiers.map((tier) => ({
  1272→        ...tier,
  1273→        price: tier.price.toString(),
  1274→      })),
  1275→    }))
  1276→
  1277→    // 5. الإرجاع
  1278→    return successResponse(safeEvents, 'تم جلب الفعاليات', {
  1279→      page,
  1280→      limit,
  1281→      total,
  1282→      totalPages: Math.ceil(total / limit),
  1283→    })
  1284→  } catch (error: unknown) {
  1285→    if (error instanceof z.ZodError) {
  1286→      return errorResponse(
  1287→        'VALIDATION_ERROR',
  1288→        'بارامترات غير صالحة',
  1289→        undefined,
  1290→        422
  1291→      )
  1292→    }
  1293→    const message = error instanceof Error ? error.message : 'خطأ غير متوقع'
  1294→    console.error('GET /api/v1/events:', message, error)
  1295→    return errorResponse('INTERNAL_ERROR', message, undefined, 500)
  1296→  }
  1297→}
  1298→
  1299→export async function POST(req: NextRequest) {
  1300→  try {
  1301→    // 1. المصادقة + الصلاحيات
  1302→    const currentUser = await getCurrentUser()
  1303→    if (!currentUser) {
  1304→      return errorResponse('UNAUTHORIZED', 'يجب تسجيل الدخول', undefined, 401)
  1305→    }
  1306→    if (currentUser.role !== 'ORGANIZER' && currentUser.role !== 'ADMIN') {
  1307→      return errorResponse('FORBIDDEN', 'غير مصرح', undefined, 403)
  1308→    }
  1309→
  1310→    // 2. التحقق من المدخلات
  1311→    const body = await req.json()
  1312→    const validated = eventCreateSchema.parse(body)
  1313→
  1314→    // 3. الإنشاء
  1315→    const event = await db.event.create({
  1316→      data: {
  1317→        ...validated,
  1318→        organizerId: currentUser.id,
  1319→        slug: validated.slug || generateSlug(validated.title),
  1320→      },
  1321→    })
  1322→
  1323→    return successResponse(event, 'تم إنشاء الفعالية', undefined, 201)
  1324→  } catch (error: unknown) {
  1325→    if (error instanceof z.ZodError) {
  1326→      return errorResponse(
  1327→        'VALIDATION_ERROR',
  1328→        'بيانات غير صالحة',
  1329→        error.errors[0]?.path[0]?.toString(),
  1330→        422
  1331→      )
  1332→    }
  1333→    const message = error instanceof Error ? error.message : 'خطأ غير متوقع'
  1334→    console.error('POST /api/v1/events:', message, error)
  1335→    return errorResponse('INTERNAL_ERROR', message, undefined, 500)
  1336→  }
  1337→}
  1338→
  1339→function generateSlug(title: { ar: string; en: string }): string {
  1340→  return title.en
  1341→    .toLowerCase()
  1342→    .replace(/[^a-z0-9]+/g, '-')
  1343→    .replace(/^-+|-+$/g, '')
  1344→}

  1347→### 8.4 Zod Schema قياسية
  1348→
  1349→```typescript
  1350→// src/lib/validators/event-schema.ts
  1351→import { z } from 'zod'
  1352→
  1353→export const eventCreateSchema = z.object({
  1354→  title: z.object({
  1355→    ar: z.string().min(3, { message: 'العنوان بالعربية قصير جداً' }),
  1356→    en: z.string().min(3, { message: 'العنوان بالإنجليزية قصير جداً' }),
  1357→  }),
  1358→  description: z.object({
  1359→    ar: z.string().max(2000, { message: 'الوصف طويل جداً' }),
  1360→    en: z.string().max(2000, { message: 'الوصف طويل جداً' }),
  1361→  }),
  1362→  startDate: z.string().datetime({ message: 'تاريخ البداية غير صالح' }),
  1363→  endDate: z.string().datetime({ message: 'تاريخ النهاية غير صالح' }),
  1364→  categoryId: z.string().cuid({ message: 'معرّف الفئة غير صالح' }),
  1365→  slug: z
  1366→    .string()
  1367→    .min(3)
  1368→    .max(100)
  1369→    .regex(/^[a-z0-9-]+$/, { message: 'الـ slug يجب أن يكون أحرف صغيرة وأرقام وشرطات' })
  1370→    .optional(),
  1371→  ticketTiers: z
  1372→    .array(
  1373→      z.object({
  1374→        name: z.object({
  1375→          ar: z.string().min(2),
  1376→          en: z.string().min(2),
  1377→        }),
  1378→        price: z
  1379→          .string()
  1380→          .refine((val) => {
  1381→            const num = parseFloat(val)
  1382→            return !isNaN(num) && num >= 0 && num < 1000000
  1383→          }, { message: 'السعر غير صالح' }),
  1384→        quantity: z.number().int().min(1).max(100000),
  1385→      })
  1386→    )

  1401→export interface SafeUser {
  1402→  id: string
  1403→  clerkId: string
  1404→  email: string
  1405→  name: string
  1406→  role: 'USER' | 'ORGANIZER' | 'ADMIN'
  1407→  isActive: boolean

  1410→export async function getCurrentUser(): Promise<SafeUser | null> {
  1411→  try {
  1412→    const { userId } = await auth()
  1413→    if (!userId) return null
  1414→
  1415→    const clerkUser = await currentUser()
  1416→    if (!clerkUser) return null
  1417→
  1418→    // مزامنة مع قاعدة البيانات
  1419→    const dbUser = await db.user.findUnique({
  1420→      where: { clerkId: userId },

  1427→          clerkId: userId,
  1428→          email: clerkUser.emailAddresses[0]?.emailAddress ?? '',
  1429→          name: `${clerkUser.firstName ?? ''} ${clerkUser.lastName ?? ''}`.trim(),
  1430→          role: 'USER',
  1431→        },
  1432→      })
  1433→      return newUser
  1434→    }
  1435→
  1436→    return dbUser
  1437→  } catch (error) {

  1439→    return null
  1440→  }

  1446→    redirect('/login')
  1447→  }
  1448→  return user
  1449→}

  1451→export async function requireRole(...roles: SafeUser['role'][]): Promise<SafeUser> {

  1599→### 8.8 API Client للواجهة
  1600→
  1601→```typescript
  1602→// src/lib/api-client.ts
  1603→interface ApiResponse<T> {
  1604→  success: boolean
  1605→  data?: T
  1606→  message?: string
  1607→  error?: { code: string; message: string; field?: string | null }
  1608→  meta?: { page: number; limit: number; total: number; totalPages: number }
  1609→}
  1610→
  1611→class ApiClient {
  1612→  private baseUrl: string
  1613→
  1614→  constructor() {
  1615→    this.baseUrl = process.env.NEXT_PUBLIC_APP_URL ?? ''
  1616→  }
  1617→
  1618→  private async request<T>(
  1619→    path: string,
  1620→    options: RequestInit = {}
  1621→  ): Promise<ApiResponse<T>> {
  1622→    try {
  1623→      const response = await fetch(`${this.baseUrl}/api/v1${path}`, {
  1624→        ...options,
  1625→        headers: {
  1626→          'Content-Type': 'application/json',
  1627→          ...options.headers,
  1628→        },
  1629→        // تأكد من إرسال cookies (للمصادقة)
  1630→        credentials: 'include',
  1631→      })
  1632→
  1633→      const data = await response.json()
  1634→
  1635→      if (!response.ok || !data.success) {
  1636→        throw new Error(data.error?.message ?? 'Request failed')
  1637→      }
  1638→
  1639→      return data
  1640→    } catch (error) {
  1641→      console.error(`API ${path} error:`, error)
  1642→      throw error
  1643→    }
  1644→  }
  1645→
  1646→  get<T>(path: string) {
  1647→    return this.request<T>(path)
  1648→  }
  1649→
  1650→  post<T>(path: string, body: unknown) {
  1651→    return this.request<T>(path, {
  1652→      method: 'POST',
  1653→      body: JSON.stringify(body),
  1654→    })
  1655→  }
  1656→
  1657→  patch<T>(path: string, body: unknown) {
  1658→    return this.request<T>(path, {
  1659→      method: 'PATCH',
  1660→      body: JSON.stringify(body),
  1661→    })
  1662→  }
  1663→
  1664→  delete<T>(path: string) {
  1665→    return this.request<T>(path, { method: 'DELETE' })
  1666→  }
  1667→}
  1668→
  1669→export const apiClient = new ApiClient()
  1670→```

  1981→### 9.2 Payment Integration — مبادئ آمنة

  1983→1. **لا تثق بالواجهة أبداً** — كل التحقق في الخادم.
